### Task 4: ✅ 设计微信文件传输助手收集方案

#### 需求分析

**用户需求**: 主动自动收集微信文件传输助手的信息内容（文件、文字、图片、视频号链接、公众号文章链接）

**技术挑战**:
1. 微信本地数据库可能加密
2. 需要root/越狱权限
3. 隐私考虑

**解决方案**: **手动导出 + 自动分析**

#### 工作流程

```
┌─────────────────┐     ┌─────────────────┐     ┌─────────────────┐

│  微信内容到文本 │────▶│  导出文件并解析 │────▶│  提取有价值信息 │
│  文件           │     │  保存 to DB     │     │  纳入舆情系统   │
└─────────────────┘     └─────────────────┘     └─────────────────┘
```

## 七、技术亮点

### 1. 渐进式实现策略

由于Python无法直接调用LLM/TDX MCP，采用**渐进式实现**：
1. Python生成提示词文件
2. 自动化任务读取提示词文件
3. 自动化任务调用LLM/TDX MCP
4. 自动化任务将结果写入文件
5. Python读取结果文件并处理

### 2. 英文RSS解析（无需feedparser）

使用Python标准库`xml.etree.ElementTree`实现RSS解析，无需安装第三方库。

### 3. 微信内容采集（手动导出+自动分析）

平衡了**技术可行性**和**用户隐私**，采用手动导出方案。
