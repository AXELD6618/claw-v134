        if attempt < MAX_RETRY:

### ✅ 2. 看门狗监控机制 (V13.4.3)
**文件**: `cloud_watchdog.py` (新创建)

#### 功能特性：
1. **连续失败跟踪** — 每个步骤独立跟踪连续失败次数
2. **自动重置** — 成功后自动重置失败计数
3. **智能告警** — 连续失败 ≥ 3 次触发 PushPlus 告警
4. **告警抑制** — 1 小时内不重复告警（避免刷屏）
5. **集成 Guardian** — 健康检查后自动运行看门狗检查

#### 看门狗状态文件：
- 路径: `cloud_state/watchdog_state.json`
- 内容:
  ```json
  {
    "steps": {
      "T0": {"consecutive_failures": 0, "last_success": "...", "last_status": "ok"},
      "T1": {...},
      ...
    },
    "last_alerts": {"T0": "2026-06-25T22:00:00", ...}
  }
  ```

#### 告警流程：
1. 步骤失败 → `record_failure(step, error)`
2. 连续失败 ≥ 3 → 触发告警
3. 发送 PushPlus 微信通知
4. 记录告警时间（`last_alerts`）
5. 1 小时内抑制重复告警
